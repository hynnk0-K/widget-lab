# 현대차 안전관제 시스템 — 통합 워크플로우 설명서

> **전체 워크플로우 흐름 / 모든 메뉴 / 역할별 진입·동작**을 하나로 정리한 문서입니다.
> 시각 자료는 FigJam 보드 2종(① 전체 메뉴 통합 플로우, ② 역할 드릴다운 플로우)과 함께 보면 됩니다.

---

## 1. 한눈에 보는 전체 흐름

사용자가 로그인하면 **역할·권한·스코프**가 판정되고, 그 결과에 따라 **진입 화면과 보이는 데이터 범위**가 달라집니다. 핵심은 — 모든 역할이 *동일한 경험 구조*(대시보드 → 뷰 전환 → 드릴다운 → 알람 처리)를 공유하되, **데이터 범위만 스코프로 필터링**된다는 점입니다.

모니터링 중 임계 초과나 위험이 감지되면 알람이 발생하고, **자동 통보 → 처리(인지·조치·완료) → 필요 시 사고 등록**으로 이어집니다.

```
로그인 → 역할 판정 → [역할별 진입 대시보드] → 뷰 전환(지도/평면도/카드/테이블/차트)
        → 드릴다운(시설 → P&ID → 센서 트렌드) → [임계 초과/위험] → 알람
        → 자동 통보 → 처리(인지·조치·완료) → 사고 등록
```

---

## 2. 워크플로우 단계별 설명

1. **로그인** — 사번/비밀번호 인증(모바일 only).
2. **역할·권한·스코프 판정** — hmg-admin 연계. 이 단계가 이후 진입점과 가시 범위를 모두 결정.
3. **역할별 진입 대시보드** — 전사/사업장 범위는 통합 대시보드, 카테고리 담당은 카테고리 대시보드, 부서·설비 담당은 더 좁은 범위로 진입.
4. **뷰 전환** — 같은 데이터를 평면도/지도 · 카드 그리드 · 테이블 · 차트로 전환해서 봄. 테이블은 CSV 내보내기 지원.
5. **드릴다운** — 평면도/카드에서 시설 클릭 → **P&ID 설비 뷰**(도면+센서 핫스팟+실시간 측정값) → 핫스팟 클릭 → **센서 트렌드 차트**(1H~30D, 임계라인 HIHI/HIGH/LOW/LOW2).
6. **알람 발생 → 처리** — 임계 초과·위험 시 실시간 알람 → 심각도별 자동 통보(SMS/Email) + 발송 이력 → 처리 팝업(인지→조치시작→조치중→완료, 사진/메모/위치) → 처리 이력 → 사고로 확대 시 인시던트 등록.
7. **현황/통계** — 시계열 비교, 통계·이상치 분석, 데이터 다운로드, 정기/사용자정의 보고서, 위험지역 히트맵.
8. **관리 → 피드백** — 마스터 계층, 임계치 프로파일, 시스템 설정. 임계치는 알람 판정에, 역할/권한은 로그인 라우팅에 반영.

## 3. 역할별 진입 & 스코프 (한 사업장 기준)

| 역할 | 진입 화면 | 보이는 범위(스코프) | 주 권한 |
|---|---|---|---|
| **시스템관리자** | 통합 대시보드 + 관리 메뉴 | 전 공장·전 카테고리 (제한 없음) | 전체 **관리**(C·R·U·D) |
| **사업장관리자** | 사업장 통합 대시보드 | 사업장 전체 (전 공장·전 카테고리) | 일부 **관리**(C·R·U·D) |
| **OT솔루션관리자** (보건팀/안전팀 등) | 담당 카테고리 대시보드 | **담당 카테고리** × 공장 내 소공장 횡단 | 카테고리 열람 + 알람 처리 |
| **현장부서조직장** (라인장/그룹장/보직관장) | 부서 대시보드 (+모바일) | 자기 공장 · 자기 부서 · 관리 하 **라인/구역**만 | 부서 범위 열람 + 알람 처리·사고 등록 |
| **OT설비/장치 현장담당자** (생산관리/보전) | 담당 설비 목록 → P&ID (+모바일) | 자기 부서 · 본인 관리 **설비/장치**만 | 담당 설비 열람 + 알람 처리·사고 등록 |

**스코프 축이 역할마다 다른 점**이 핵심입니다.
- 관리자·사업장관리자 → 범위 제한 없음(공장 전체)
- OT솔루션관리자 → **카테고리** 축으로 필터 (소공장 횡단)
- 현장부서조직장 → **조직/부서** 축으로 필터 (라인/구역)
- OT설비현장담당자 → **설비** 축으로 필터 (가장 좁음, 바로 P&ID 진입)

## 6. 피드백 루프 & 구현 연결

- **임계치 프로파일(WSI-M-010) → 알람 판정** — 관리자가 측정포인트별로 설정한 임계치(HIHI/HIGH/LOW/LOW2 등)가 실시간 알람 발생 기준으로 사용됨.
- **역할/권한(WSI-M-014/015) → 로그인 라우팅(MC-L-001)** — 권한·스코프 설정이 진입 화면과 가시 범위를 결정.

**권한 스코프 ↔ 마스터 계층 매핑** — 역할별 스코프는 마스터 데이터 계층에 그대로 대응시키면 깔끔합니다.

```
회사(Company) → 사업장(Site) → 공장(Factory) → 공정(Process) → 라인(Line) → 설비/측정포인트(Equipment/Point)
   └ 시스템관리자/사업장관리자 : 사업장 이하 전체
   └ OT솔루션관리자          : (카테고리) × 공장 이하 소공장 전체
   └ 현장부서조직장          : 공장·부서 → 관리 라인/구역
   └ OT설비현장담당자        : 담당 설비/포인트
```
---
